# PRT(Peer Review Template)
- [ ]  **1. 주어진 문제를 해결하는 완성된 코드가 제출되었나요?**
    - 최종 결과물에서 주어진 해결 과제에 따라 완성된 코드를 제출하였다.
    - 마크다운 형식의 보고서가 미흡하여 아쉽다.
    
- [ ]  **2. 전체 코드에서 가장 핵심적이거나 가장 복잡하고 이해하기 어려운 부분에 작성된 
주석 또는 doc string을 보고 해당 코드가 잘 이해되었나요?**
    - 해당 코드의 기능, 존재 이유, 작동원리를 주석을 통해 기술하였다.
    - 주석을 보고 코드 이해가 잘 되도록 하였다.
        
- [ ]  **3. 에러가 난 부분을 디버깅하여 문제를 해결한 기록을 남겼거나
새로운 시도 또는 추가 실험을 수행해봤나요?**
    - 여러가지 실험을 시도하고 문제를 해결한 기록을 남겼다.
    - 특히, 고양이 수염을 각도에 합성을 수행하여 새로운 시도와 추가 실험에 공을 많이 들인 점은 배울만한 점이다.
        
- [ ]  **4. 회고를 잘 작성했나요?**
    - 주어진 문제를 해결하는 완성된 코드 내지 프로젝트 결과물에 대해 배운점과 아쉬운점, 느낀점 등이 기록되어 않으나 다양한 각도를
      통한 실험에서 많이 학습하였음이 보였다.
        
- [ ]  **5. 코드가 간결하고 효율적인가요?**
    - 코드 중복을 최소화하고 간결하였으며, 효율적으로 작성하였다.


# 회고(참고 링크 및 코드 개선)
```
다양한 실험을 하였고 그 과정을 기록하였다는 점을 학습할 수 있어 좋았다.
```
